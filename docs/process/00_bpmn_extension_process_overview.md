# BPMN Extension Process Overview

This document describes the general workflow for proposing a BPMN extension.

## Main Flow

1. Analyse the need for extension proposal
2. Describe concepts of the BPMN extension
3. Develop BPMN extension
4. Validate and evaluate the BPMN extension
5. Check other new constructs to be introduced
6. Publicise the BPMN extension

## Process Logic

The process starts with an intention to extend BPMN.

If there is no need for extension, the process ends with: Extension not proposed.

If there is need for extension, the process continues to concept description, development, validation and publication.
