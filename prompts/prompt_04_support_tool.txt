DEFINIÇÕES QUE DEVEM SER USADAS QUANDO RELEVANTE:

Extension Available: resultado obtido quando uma extensão BPMN é incorporada a uma ferramenta de modelagem existente, como Meta4Model-BPMN, BPMN.io, Camunda Modeler ou Bizagi Modeler. A extensão fica disponível para uso sem necessidade de criar uma ferramenta nova.

Extension Applied: resultado obtido quando uma ferramenta dedicada é implementada, testada, corrigida e disponibilizada aos usuários para suportar a extensão.

CONTEÚDO ESPERADO DA RESPOSTA:

1. Decidir a Estratégia de Ferramenta
O Extender deve primeiro avaliar se ferramentas existentes, como Meta4Model-BPMN, BPMN.io, Camunda Modeler ou Bizagi Modeler, podem ser adaptadas para suportar a extensão antes de considerar uma ferramenta dedicada.

2. Adicionar Construtos a uma Ferramenta Existente
Se a estratégia for adaptar uma ferramenta existente, os construtos devem ser adicionados ou configurados nessa ferramenta. O resultado esperado é Extension Available.

3. Implementar uma Nova Ferramenta, se Necessário
Se uma ferramenta dedicada for necessária, ela deve suportar metamodelo, sintaxe concreta, criação/edição de construtos, validação, persistência, importação/exportação e integração com BPMN.

4. Testar a Ferramenta
A ferramenta deve ser testada verificando criação de construtos, renderização da sintaxe concreta, edição de propriedades, regras de validação, salvamento/carregamento e compatibilidade BPMN.

5. Corrigir Problemas
Problemas identificados devem ser corrigidos e a ferramenta deve ser testada novamente.

6. Disponibilizar a Ferramenta
Quando não houver correções pendentes, a ferramenta deve ser disponibilizada aos usuários. O resultado é Extension Applied quando uma ferramenta dedicada foi criada.

Artefatos/resultados esperados:
- Modelling tool for the extension
- Tool test results
- Corrected modelling tool
- Extension Available (ferramenta existente adaptada)
- Extension Applied (ferramenta dedicada implementada e disponibilizada)
